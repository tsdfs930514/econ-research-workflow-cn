# /logic-check — 论文逻辑检查

当用户调用 `/logic-check` 时，按以下步骤操作：

## 步骤 1：收集信息

向用户询问：

1. **语言**（必填）— `CN`（中文）或 `EN`（英文）
2. **待检查文本**（必填）— 论文文本（可以是单个章节或全文）

## 步骤 2：审查哲学

**高容忍度预设：** 假定文本已经过多轮修改。这是终稿红线审查，而非全面编辑。

**仅报错原则：** 只在遇到以下问题时才发言：
- 会让读者困惑
- 会损害论文可信度
- 造成逻辑矛盾
- 会让审稿人质疑论文严谨性

**禁止优化：** 不要评论：
- 风格偏好（"我会换个说法..."）
- 不影响含义的细微措辞选择
- 虽非最优但可接受的段落顺序
- 引用格式（除非明显错误）
- 写作质量——这不是润色步骤（润色请用 `/polish`）

## 步骤 3：审查维度

### 3.1 致命逻辑矛盾

- 一个章节中的陈述与另一个章节直接矛盾
- 对结果的声明与实际发现不符
- 引言中提出的假设未被检验，或检验方式与描述不同
- 结论不被已呈现的结果所支持

### 3.2 术语一致性

- 核心概念无故改名（如一处写"处理组"，另一处写"实验组"——指同一事物）
- 缩写定义与使用不一致
- 变量名称前后不同（如"收入"变成"所得"，但指的是同一变量）

### 3.3 严重语病

仅标记导致真正歧义或误解的语言问题：
- 因语法问题导致含义不清的句子
- 造成错误语义的悬垂修饰语
- 模糊原意的双重否定
- 不标记：轻微语法问题、错别字或风格偏好

### 3.4 经济学特有检查

**变量名/表格编号一致性：**
- 正文中的变量名与表格中的一致
- 正文引用的表格/图表编号存在且内容匹配
- "表3显示..."确实对应表3的内容

**显著性描述与表格数据匹配：**
- 正文说"在1%水平上显著"但表格显示 **（5%）→ 标记
- 正文说"不显著"但表格显示 *** → 标记
- 正文描述的系数符号与表格一致

**因果表述恰当性：**
- 在仅支持相关性的方法中使用因果语言（"X导致Y"、"X对Y的影响"）→ 标记
- 在无识别策略的描述性/OLS回归中使用"影响"或"效应"语言 → 标记
- 如果论文使用了有效的识别策略（DID、IV、RDD），因果语言是恰当的

**样本描述一致性：**
- 正文提到的 N（观测值）与表格表头一致
- 正文提到的样本期间与数据描述一致
- 子群描述在正文和表格之间一致

## 步骤 4：输出

**如果未发现问题：**

```
[检测通过，无实质性问题]

已审查逻辑矛盾、术语一致性、严重语病和经济学特定问题，
未发现需要关注的问题。
```

**如果发现问题：**

```
### 发现的问题

**1. [问题类别]：[简要描述]**
- 位置：[在文本中的位置]
- 问题：[具体错误]
- 建议：[如何修复]

**2. [问题类别]：[简要描述]**
- 位置：[在文本中的位置]
- 问题：[具体错误]
- 建议：[如何修复]
```

每个问题保持简短——最多 2-3 行。这是红线审查，不是详细评审。

问题类别：
- 逻辑矛盾
- 术语不一致
- 严重语病
- 变量/表格不一致
- 显著性描述不匹配
- 因果表述不当
- 样本描述不一致
